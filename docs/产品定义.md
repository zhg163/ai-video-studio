可以把 Flova.ai 这套流程理解成：它不是“输入一句 prompt 就吐出一个视频”，而是把传统影视制作里的编剧、分镜、美术、配音、剪辑这些环节，压缩成一个由 AI 协作完成的生产链。官方文档把它描述为从 story/script → characters, costumes & props, and storyboards → video → voiceover & music → final editing 的全流程创作。([Flova AI][1])

下面按你说的五个阶段详细拆开讲。

## 1）创意输入：把“想法”变成可执行的项目说明

这是整个流程的起点，本质上相当于影视制作里的“立项 + 创意阐述 + 脚本输入”。

Flova 支持几种输入方式：
用户可以直接输入 idea、story、script 或者具体的视频需求；也可以上传图片、音频、文本等素材，再补充创作要求；还可以直接输入完整脚本，让系统更精准地响应。官方还提到，描述越具体，输出越准确，尤其是人物特征、故事内容、画幅比例、语言、视觉风格这些信息，会直接影响后续生成质量。([Flova AI][1])

这一阶段 Flova 实际上在做三件事：

第一，**理解任务目标**。
它要先判断你到底是在做广告片、剧情短片、产品演示，还是角色动画。

第二，**补齐创意约束**。
比如人物长相、服装、道具、镜头语言、风格基调、时长、比例、旁白语言等。官方文档明确说它会智能生成 creative brief 供用户确认，这一步很像传统制作中的“导演阐述 + 制片需求单”。([Flova AI][1])

第三，**把自然语言转成结构化生产信息**。
也就是从“我想做一个赛博朋克风格的 30 秒品牌片”，转成后续 AI 能调用的镜头、角色、素材、配乐、剪辑参数。

所以，“创意输入”不是简单地输入一句话，而是把模糊创意压缩成一个**可生产的项目蓝图**。

---

## 2）代理分工：把一个大任务拆给不同 AI 角色

这一步是 Flova 和很多普通视频生成工具最大的差异之一。第三方介绍把它概括为“多智能体协作系统”或“类似影视团队的 AI Agent 机制”，也就是不是单模型从头干到尾，而是按环节分工。([AI星踪岛][2])

虽然官方没有把所有 agent 名字逐个列出来，但从文档和产品描述能看出，它的任务拆分大致对应传统影视团队里的这些岗位：

* **编剧/策划代理**：理解主题、扩写故事、整理脚本
* **分镜代理**：把脚本拆成 shot、scene、storyboard
* **视觉生成代理**：负责角色、道具、场景、关键帧、视频镜头生成
* **音频代理**：处理配音、音乐、音效
* **剪辑代理**：负责时间线装配、节奏、片段衔接、导出

官方文档提到 Flova 会“intelligently plan the creation process and provide next-step suggestions”，这说明它不是被动等用户逐步下指令，而是会主动规划下一步该做什么。它还支持 skills 和 custom workflows，表示这套“代理分工”其实可以通过默认技能和自定义工作流去调度。([Flova AI][1])

你可以把这一层理解为一个“AI 制片中台”：

* 上层接收用户创意
* 中层做任务拆解与顺序编排
* 下层调用不同模型完成具体生产

因为 Flova 集成了多种图像、视频、音频模型，系统还会根据任务类型去选更合适的模型。官方列出的可选模型包括 Nano Banana Pro、Kling 3.0、Sora 2 Pro、Veo 3.1、Suno v5 等。([Flova AI][1])

所以这一阶段的核心，不是“生成内容”，而是**先把任务拆对、把工种分清、把模型选对**。

---

## 3）视觉与音频生成：把脚本和分镜真正产出成素材

这是“出片”的核心阶段，对应传统制作里的美术、摄影、录音、配乐。

### 3.1 视觉生成

根据官方描述，Flova 的视觉链路包括：

* 角色
* 服装与道具
* 分镜
* 关键帧
* 视频镜头

而且这些不是一次性黑盒输出，而是可以通过 storyboard、shot、keyframe、video 四个层级分别修改。比如你可以：

* 在 storyboard 层修改某个镜头方案
* 在 shot 层增删镜头
* 在 keyframe 层微调视觉内容
* 在 video 层直接改视频生成方式，如 text-to-video、first-frame-to-video、start-and-end-frame-to-video ([Flova AI][1])

这说明 Flova 的视觉生成是**分层可编辑**的，不是“全片重生”。

另外，第三方资料反复强调它的一个卖点是**角色一致性**，尤其是在 360° 机位变化时尽量保持人物面部和服装不漂移。这个能力对叙事类视频尤其重要，因为很多普通 AI 视频工具一旦切镜，人物容易“换脸”或服装变形。([Pollo AI][3])

### 3.2 音频生成

官方在入门文档中直接把 voiceover & music 放进了主流程，且可选音频模型包含 Suno v5；中文文档还说明支持音乐生成和音频生成，并支持通过 `@` 引用图片、音频、视频等素材作为参考。([Flova AI][1])

这意味着音频层至少包含：

* 旁白/配音
* 背景音乐
* 与视觉素材的参考联动

从生产逻辑上看，Flova 是在同一个项目上下文里同时管理视觉和声音，而不是让用户先去别的平台做音乐，再导回来。

所以这个阶段的本质是：
**把抽象创意转成一组可剪辑的“镜头素材 + 音频素材”资产包。**

---

## 4）剪辑合成：把素材装进时间线，形成成片

这一步对应传统后期制作。

官方文档写得比较明确：Flova 支持 “Assemble Videos and Generate Timeline”，并且允许手动编辑和导出。可进行的编辑包括：

* 拖拽裁剪片段边缘
* 调整播放速度
* 修改音量
* 右键对片段进行重新生成、手动编辑、替换素材、删除
* 通过自然语言下达剪辑要求，比如“裁到某个时间段” ([Flova AI][1])

这说明 Flova 的“剪辑合成”不是简单拼接，而是具备一个轻量到中度的非线性编辑逻辑，接近“时间线 + 对话式编辑”的结合。

第三方资料也把它描述为“专业级时间轴自动化”或“可编辑时间轴”，强调能自动做转场、音乐、节奏调整，同时允许像 Premiere 那样细调每个片段。([Pollo AI][3])

这一阶段通常包含四个核心动作：

1. **镜头装配**
   把前面生成的多个 shot 排进 timeline。

2. **节奏控制**
   决定哪些镜头快切，哪些镜头停留更久，是否加转场。

3. **声画同步**
   让配音、音乐、画面节奏相互对齐。

4. **成片输出**
   官方支持导出最终视频，也支持导出 PR 项目文件，便于继续在专业剪辑软件中精修。([Flova AI][1])

所以这里的关键，不是再“生成一次”，而是**把前面生产出来的资产组织成可观看的叙事结构**。

---

## 5）用户迭代：通过对话和局部编辑不断修正结果

这是 Flova 另一个很重要的设计思想。官方多次强调它支持“通过对话持续创作”，并且会根据生成结果主动给出下一步建议。用户确认后，它再继续推进。([Flova AI][1])

这和传统 AI 视频产品“一次 prompt → 一次结果”的思路不同。Flova 更像一个可反复打磨的项目环境，迭代方式包括：

### 5.1 对话式迭代

你可以直接说：

* 这个镜头再电影感一点
* 把角色衣服改成黑色风衣
* 音乐更紧张一点
* 结尾镜头拉长 2 秒

系统会在已有上下文中理解这些修改，而不是要求你重新从头写完整 prompt。([Flova AI][1])

### 5.2 分镜级迭代

官方支持在 storyboard 中逐项编辑模型、prompt、参考图、生成方法、分辨率、时长等。([Flova AI][1])
这意味着用户可以像改拍摄分镜一样改 AI 生成计划。

### 5.3 镜头级迭代

可以对单个 shot 做替换、增删、重生，而不影响整个项目。([Flova AI][1])

### 5.4 偏好记忆与结果筛选

当一次生成出现多个结果时，用户可以标记 favorite，Flova 会优先吸收这些偏好。官方把它表述为会优先纳入你“liked”的内容。([Flova AI][1])

### 5.5 项目版本分支

高级功能里还有 “Start New Project” 和 “Return to This Moment”。前者保留当前上下文去开新项目，适合连续剧、多集内容；后者则回退到某个节点，清除后续无效结果。([Flova AI][4])

这实际上已经很接近影视制作里的“版本管理”与“创作分支”。

---

## 一句话总结这五步的本质

Flova 的这五个阶段，本质上是在用 AI 重建一条简化版影视工业流水线：

**创意输入**：把想法说清楚
**代理分工**：把任务拆成编剧、分镜、视觉、音频、剪辑等模块
**视觉与音频生成**：分别产出画面和声音素材
**剪辑合成**：把素材组装成完整视频
**用户迭代**：围绕局部结果持续修改，直到成片定稿 ([Flova AI][1])

# 一、产品定义

这个产品本质上不是“文生视频工具”，而是一个：

> **面向内容创作与企业生产的 AI 多智能体视频制作平台**

它融合两类能力：

**1. OpenCode 能力**
偏“AI 生产中台 / Agent 编排 / 工作流执行 / Tool 调度 / 会话与记忆”

**2. 视频能力**
偏“脚本生成 / 分镜 / 角色设定 / 图片生成 / 视频生成 / 配音 / 音乐 / 剪辑合成”

所以国产版 Flova 不应做成一个单点应用，而应做成三层产品：

* **创作层**：用户输入创意，编辑分镜、镜头、时间线
* **编排层**：多 Agent 拆解任务、调度模型、执行工作流
* **生成层**：接入国产图像/视频/音频模型，输出成片

# 四、核心业务流程

## 主流程

```text
用户输入创意
→ 创意理解 Agent
→ 生成脚本
→ 分镜 Agent 拆镜头
→ 角色/场景/风格抽取
→ 图像生成 Agent 出关键帧
→ 视频生成 Agent 逐镜头出视频
→ 音频 Agent 生成配音/音乐
→ 剪辑 Agent 自动拼接
→ 用户在时间线编辑
→ 局部重生成
→ 导出视频
```

这个流程的关键不是“生成一次完成”，而是：

> **每个阶段都必须可回退、可替换、可局部重算**

这点决定你系统是不是“工具”，还是“生产平台”。

## 2）创意输入模块 Creative Brief

输入方式支持：

* 文本输入
* 上传图片/视频作为参考
* 上传脚本
* 上传品牌资料 / 产品说明 / 课程内容

系统在这里做两件事：

### a. 创意结构化

把自然语言转成：

* 视频目标
* 受众人群
* 视频长度
* 风格
* 语言
* 画幅比例
* 角色设定
* 场景设定
* 输出平台（抖音 / B站 / 视频号 / 企业内训）

### b. 生产约束抽取

比如：

* 必须出现品牌 Logo
* 必须有某个角色
* 必须是中文旁白
* 不能用某种视觉风格
* 时长必须 60 秒内

这一步非常像你做 AI 系统里的“结构化意图识别”。

# 十二、关键难点与风险

## 1. 角色一致性

这是视频类产品最难的点之一。
解决思路：

* 角色设定图固定
* 多镜头统一角色 embedding / reference
* 强约束 prompt 模板
* 优先首帧/首尾帧视频生成

## 2. 任务耗时长

视频生成天然慢。
解决思路：

* 全异步
* 队列可见
* 分阶段返回结果
* 单镜头先预览低清，后生成高清

## 3. 成本高

GPU 和视频 API 非常烧钱。
解决思路：

* 分层计费
* 候选数量限制
* 低清预览
* 热门模板缓存
* 失败自动重试但限制次数

## 4. 输出不稳定

解决思路：

* Prompt 模板化
* 强约束数据结构
* 模型路由策略
* 审核 Agent 二次把关

## 5. 合规与版权

解决思路：

* 关键词审查
* 输出内容检测
* 素材来源标记
* 企业库与公共库隔离
