# 开发任务拆解

## 1. 文档目标

本文用于将 `ai-video-studio` MVP 开发计划进一步拆解为可执行任务项，便于研发排期、分工、跟踪和验收。

## 2. 拆解原则

1. 以业务闭环为主线拆解，不按技术兴趣拆解。
2. 每个任务必须有明确输入、输出和完成标准。
3. 优先拆到可交付的模块级和接口级任务。
4. 后端、前端、测试、运维任务同步列出。
5. 任务设计优先支持并行执行，但不打乱主链路依赖。

## 3. 总任务视图

MVP 总任务拆为 10 个工作包：

1. 基础工程与环境
2. 项目中心
3. Brief 模块
4. Script 模块
5. Storyboard 模块
6. 媒体生成模块
7. 任务中心与异步系统
8. Timeline 模块
9. Render 模块
10. 稳定性、测试与运维

## 8. 完成定义

每个任务项完成时必须满足：

1. 代码已提交并通过评审。
2. 接口文档已同步。
3. 必要的数据结构已落库。
4. 至少有基础测试。
5. 前后端联调通过。
6. 可被纳入阶段验收。

## 9. 拆解结论

MVP 执行中最重要的不是把任务拆得更细，而是确保所有任务始终服务于主闭环：

1. 创意到分镜
2. 分镜到素材
3. 素材到成片

只要任务拆解持续围绕这三段推进，就能避免需求扩散和资源分散。


下一步可选方向

Shot 媒体生成 — 接入图片/视频生成 API（GPT Image 1、Kling 3.0），实现 shots.py 的 generate_keyframe/generate_video 端点
Timeline 模块 — 将 Storyboard 的 shots 组装成 Timeline（时间轴编排）
TTS/配音模块 — 接入 Qwen3-TTS，生成旁白音频
FFmpeg 渲染 — media_render_service 的成片导出
E2E 全链路测试 — 从创意输入到 Storyboard 确认的端到端集成测试（不 mock LLM）
