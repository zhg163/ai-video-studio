下面直接给你一套可落地的 **企业级 MVP 架构蓝图 + 模块清单 + 表结构 + 接口设计**。
我按你常用的技术栈来设计：**FastAPI + Redis Stack + MongoDB + PostgreSQL/MySQL + MinIO + 异步任务队列**。

---

# 一、MVP 目标定义

先明确这版 MVP 不追求“全功能 Flova”，而追求：

> **从创意输入到成片导出的最小可用闭环**

## MVP 闭环能力

1. 用户创建项目
2. 输入创意 / 脚本 / 参考素材
3. AI 生成 creative brief
4. AI 生成脚本
5. AI 自动拆分 storyboard / shot
6. 单镜头生成关键帧图
7. 单镜头生成视频片段
8. 生成配音 / 字幕
9. 自动拼接时间线
10. 导出成片
11. 支持单镜头重生成
12. 支持项目版本记录

# 四、核心流程蓝图

---

## 4.1 主工作流

```text
创建项目
→ 输入创意
→ 生成 Brief
→ 生成脚本
→ 拆分 Storyboard
→ 生成每个 Shot 的关键帧
→ 生成每个 Shot 的视频
→ 生成旁白 / 字幕 / BGM
→ 自动拼接 Timeline
→ 用户微调
→ 导出成片
```

---

## 4.2 Shot 级重生成流程

```text
用户选择某个 shot
→ 修改 prompt / 模型 / 时长 / 参考图
→ workflow-service 创建重生成任务
→ media-service 调模型生成新素材
→ 更新 shot 最新候选版本
→ timeline 引用新素材
→ 保留旧版本可回退
```

---

## 4.3 导出流程

```text
用户点击导出
→ render-service 锁定当前 timeline 版本
→ 拉取视频轨 / 音频轨 / 字幕轨
→ FFmpeg 合成
→ 上传 MinIO
→ 写入 render_task 与导出地址
→ 回写项目最新成片信息
```


```text
draft
→ brief_ready

→ storyboard_ready
→ generating_assets

→ archived
```

异常分支：

```text
任意状态 → failed

```

```text
pending
→ image_generating
→ image_ready

→ video_ready
```

```text
image_generating → failed
video_generating → failed
```

# 八、接口设计

下面给你一套 **企业级 REST API 草案**，MVP 足够用了。

---

## 8.1 项目接口

### 创建项目

`POST /api/v1/projects`

请求：

```json
{
  "project_name": "AI产品宣传片",
  "description": "面向企业客户的视频",
  "aspect_ratio": "16:9",
  "language": "zh-CN",
  "duration_target_sec": 60
}
```

响应：

```json
{
  "code": 0,
  "data": {
    "project_id": 1001,
    "status": "draft"
  }
}
```

---

### 查询项目列表

`GET /api/v1/projects?page=1&page_size=20`

---

### 查询项目详情

`GET /api/v1/projects/{project_id}`

---

### 归档项目

`POST /api/v1/projects/{project_id}/archive`

---

## 8.2 创意输入接口

### 提交 creative brief 原始输入

`POST /api/v1/projects/{project_id}/brief`

请求：

```json
{
  "input_text": "做一个60秒的AI视频平台宣传片，风格科技感，中文旁白",
  "reference_asset_ids": [3001, 3002]
}
```

响应：

```json
{
  "code": 0,
  "data": {
    "brief_version_no": 1,
    "workflow_task_id": 9001,
    "status": "processing"
  }
}
```

---

### 获取最新 brief

`GET /api/v1/projects/{project_id}/brief/latest`

---

### 手工更新 brief

`PUT /api/v1/projects/{project_id}/brief/{version_no}`

---

# 十、异步任务与队列设计

MVP 强烈建议全部生成类操作走异步。

---

## 10.1 队列分类

### 高优先级

* brief_generate
* script_generate
* storyboard_generate

### 中优先级

* image_generate
* voice_generate
* subtitle_generate

### 低优先级

* video_generate
* render_export
* bgm_generate

---

## 10.2 Worker 分类

### llm-worker

处理：

* brief
* script
* storyboard
* prompt rewrite

### image-worker

处理：

* 关键帧生成
* 分镜图生成

### video-worker

处理：

* shot 视频生成

### audio-worker

处理：

* tts
* bgm
* subtitle alignment

### render-worker

处理：

* ffmpeg 合成
* 导出
* 封面抽帧

---

## 10.3 幂等设计

每个生成请求要带幂等键：

```text
tenant_id + project_id + biz_ref_id + task_type + hash(input_payload)
```

避免重复点击造成重复烧 GPU。
